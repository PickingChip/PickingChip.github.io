### 特殊符号

`const`常量限定符：

- const 修饰的变量初始化后不允更改，生命周期为整个程序周期，可以直接在程序中用变量名替换对应的值，和宏相似但是宏没有数据类型。
- const 修饰的全局变量链接性为内部，通常变量禁止在头文件中声明以防止重复定义，但是`const` 修饰的变量可以在头文件中声明不会引发冲突 ，除非显示在前添加`extern`修饰。
- 指针常量与常量指针

`volatile`易变限定符：告诉编译器这个变量可能会被硬件修改，不要对他进行优化，每次读取时从内存重新读取。

`mutable`可变限定符：即使是`const`修饰的对象，如果成员被`mutable`修饰也可改变。

`explicit`函数说明符：修饰构造函数，表明函数不接受隐式转换。

`auto`类型说明符：之前用于说明变量存储在内存中，C++11之后改为自行推断变量的类型。



### 初始化方式

C++有三种初始化方式：直接初始化，拷贝初始化，列表初始化。

- 使用`=`是拷贝初始化，先创建一个临时对象再拷贝到目标对象，对于类对象这个过程中会出现隐式转换。
- 使用`()`是直接初始化，对于基本类型等同于`=`，但是对于类直接调用构造函数进行初始化不涉及隐式转换。
- 使用`{}`是列表初始化，会进行窄化转换检查，对于类不涉及隐式转换。

内联

内联函数通过增大代码体积减少函数调用时的损耗，适用于较为短小且被经常调用的函数。相较于宏的文本替换，内联函数传参是值的传递，会对传参类型进行检查，不能递归调用。

内联函数可以不遵守单一规定原则，可以在头文件中定义。`inline` 只是对编译器的建议，函数太长、递归等不合理的情况下编译器有权忽略。



### 智能指针

智能指针能自动消除所指向动态分配的内存空间，防止内存泄漏，使用时需要引用`<memory>`文件，有以下三种类型：

- unique_ptr：一个指针独占资源所有权，不可拷贝，单独一个指针负责资源的管理。
- shared_ptr：所有指针共享资源所有权，有引用计数，每一个指针计数一次，计数为零时资源释放。
- weak_ptr：所有指针共享资源所有权，但是不计数，用于处理循环引用的情况。

```c++
/* 创建智能指针 */
std::unique_ptr<int> ptr1(new int);  //不能使用=进行初始化
/* C++14起推荐使用make_unique，但是不支持数组 */
auto ptr2 = std::make_unique<int>();
/* 转让所有权 */
auto ptr3 = std::move(p1);
/* 返回智能指针当中的原始指针 */
ptr1.get();
```



### Lambda 表达式

Lambda 表达式是 C++11 引入的一种匿名函数，格式如下：

```c++
/* [捕获列表](参数列表) -> 返回类型 { 函数体 } */ 
```

- 参数列表用于调用 Lambda 表达式时的参数传递。
- 返回类型可以不给出让编译器自行推导，没有用到参数列表可以省略。
- 每个lambda都有唯一的匿名类型，只能用`auto`或`std::function`存储，不能相互赋值。
- 捕获列表用于决定如何访问外部作用域的变量，一般有值捕获和引用捕获，值捕获变量相当于`const`修饰不能被改变，使用 mutable 关键字才能改变。

```C++
int x, y = 10;
auto f1 = [&x, y] { x = y; std::cout << ++x + y<<endl; }; //值捕获x，引用捕获y
auto f2 = [&](double z)mutable -> double  { x = ++y;  return(x + y)*z; };//[&]全部引用捕获，[=]全部值捕获
f1();//调用函数
cout<<f2(3.14)<<endl;
```



### 函数包装器

函数包装器（`std::function`）是C++11引入的一个通用多态函数封装器，用于存储，复制和调用任何可调用对象，格式如下：

```c++
/* std::function<返回类型(参数类型列表)> 变量名; */
```

- 使用函数包装器时需要引用`<functional>`文件。
- 函数包装器性能较差，难以内联。
- 函数指针无法调用类的成员函数（成员函数指针可以）与 lambda 表达式，函数包装器可以通过`bin()`绑定对象与成员函数。

```C++
#include <functional>
#include <iostream>

class Calculator {
public:
    int multiply(int a, int b) {
        std::cout << "multiply(" << a << ", " << b << ")\n";
        return a * b;
    }
};

void hello() {
    std::cout << "Hello, world!\n";
}

int main(){
    using namespace std::placeholders; //用于使用_1，_2占位符
    Calculator calc;
    std::function<void()> func1 = hello;//存储普通函数
    std::function<int(int,int)> func2 = [](int a, int b)->int{return a+b;};//存储lambda
    /* 存储类的成员函数，bind()函数将类的成员函数与对象绑定同时指定绑定的参数 */
    std::function<int(int, int)> func3 = std::bind(&Calculator::multiply, &calc, _1, _2);//推荐使用lambda表达式捕获 calc 进行函数调用

    func1();
    std::cout<<func2(4,5)<<std::endl;
    std::cout<<func3(2,3)<<std::endl;
    return 0;
}
```



### 重载

函数重载的关键是函数的参数列表——也称为函数特征标，使用上下文确定调用哪个函数。

编译器在检查函数特征标时， 将把类型引用和类型本身视为同一个特征标。

在C++函数重载中，普通变量（值传递）的 `const` 属性不被视为可以区分重载函数的依据，但指针或引用的 `const` 属性可以

### 类

类的成员默认是私有的，而结构体默认是公开的。

定义在类声明中的函数自动成为内联函数。定义在类声明之外的函数使用作用域解析符进行标识。

声明为`const`的类对象不能调用正常的成员函数，因为普通成员函数可能修改对象。但是可以调用 const 成员函数。在函数名后加 `const` 相当于告诉编译器这个函数不会修改对象（除了 mutable 成员）。
